Localization Spreadsheet JSONify for LDSTech

                To use this app:
                
                First, create a localization spreadsheet that looks like this one.
                
                Make sure the spreadsheet you want to use is public or that you are logged in
                and have at least view access to it.
                
                Then, simply call 
                with the following querystring parameters set:
                
                spreadsheet_id   required       ex: 0AuWQ-iCrlReTdE9YeVJhdFk0NW4zZlpRZTYtQzN3a2c
                sheet_name       required       ex: Main
                threshold        optional       ex: 0.8
                callback         optional       ex: languageLoader
                
                Try it out:
                

                    
                

                
                Parameters:
                
                spreadsheet_id
                
                The key parameter from a Google Spreadsheet URL, e.g.
                "https://docs.google.com/spreadsheet/ccc?key=0AuWQ-iCrlReTdE9YeVJhdFk0NW4zZlpRZTYtQzN3a2c"
                
                sheet_name
                
                The sheet name to use inside of the spreadsheet.
                
                threshold
                
                Controls the percentage of fully-translated strings a translation must have
                before it is deployable. In other words, if you have 10 total strings, and
                your threshold is 0.8, then at least 8 of the strings in a translation must
                be set, or the language is not considered deployable.
                
                callback
                
                JSONP callback function name, which can then load the translations into the
                current page.
                
                For example, this URL, will generate a JSON string translation table using this spreadsheet.
                
                The output from the script will be a hashtable/dictionary with keys for each file listed in the file column.
                Each file has metadata information for each key and a language_codes section listing the translations of keys
                in that particular file.

                If you have country-specific translations, it is recommended that you add them
                to the table after non-country-specific base translations. In other words, if
                you have a translation for "pt" and one for "pt_PT", put the "pt_PT" translation
                in a column to the right of "pt". The localization script will apply the base
                translation to unfilled cells in the country-specific translation.  And these
                base translations will count towards the completion threshold.
